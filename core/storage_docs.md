# Настройка хранилищ с обработкой файлов для Yandex Cloud

## Описание

Реализована система загрузки медиафайлов (изображений и видео) с предварительной обработкой на сервере перед загрузкой в Yandex Object Storage.

## Архитектура

1. **Загрузка файла** → Файл сохраняется во временную директорию `media_temp/`
2. **Обработка** → Файл обрабатывается (сжатие, водяной знак)
3. **Загрузка в облако** → Обработанный файл загружается в Yandex Cloud
4. **Очистка** → Временные файлы удаляются с сервера

## Структура файлов

```
core/
├── storage_backends.py    # Базовое хранилище с обработкой
├── image_storage.py       # Хранилище для изображений
├── video_storage.py       # Хранилище для видео
├── storage.py             # Утилиты для получения хранилищ
├── apps.py                # Инициализация хранилищ
└── storage_docs.md        # Эта документация
```

## Обработка изображений

При загрузке изображения выполняются следующие операции:

1. **Конвертация формата**:
   - PNG, GIF, WEBP → JPEG
   - Удаление прозрачности (замена на белый фон)

2. **Изменение размера**:
   - Максимальный размер: 800x600 пикселей
   - Сохранение пропорций

3. **Сжатие**:
   - Качество: 85%
   - Оптимизация файла

4. **Водяной знак**:
   - Добавление логотипа в правый нижний угол
   - Прозрачность: 30%
   - Размер: 1/4 от ширины изображения

## Обработка видео

При загрузке видео выполняются следующие операции:

1. **Сжатие**:
   - Видеокодек: H.264 (libx264)
   - Битрейт: 2500k
   - Аудиокодек: AAC
   - Аудиобитрейт: 128k

2. **Водяной знак**:
   - Добавление логотипа в правый нижний угол
   - Прозрачность: 70%
   - Размер: 1/4 от ширины видео

3. **Оптимизация для веба**:
   - Флаг: +faststart
   - Формат: yuv420p

## Настройка

### 1. Переменные окружения

Создайте файл `.env` на основе `.env.yandex.example`:

```bash
USE_YANDEX_CLOUD=True
YANDEX_CLOUD_ACCESS_KEY_ID=your_access_key_id
YANDEX_CLOUD_SECRET_ACCESS_KEY=your_secret_access_key
YANDEX_CLOUD_BUCKET_NAME=your_bucket_name
YANDEX_CLOUD_ENDPOINT_URL=https://storage.yandexcloud.net
```

### 2. Настройка Yandex Cloud

1. Создайте бакет в Yandex Object Storage
2. Получите IAM-токен или создайте сервисный аккаунт с правами на запись
3. Получите Access Key и Secret Key

### 3. Установка зависимостей

```bash
pip install django-storages boto3 pillow moviepy opencv-python
```

### 4. Проверка настройки

```bash
python manage.py check
```

## Модели, использующие кастомные хранилища

- `Event.image` - Главное изображение мероприятия
- `Event.video_url` - Видео мероприятия
- `EventImage.image` - Дополнительные фотографии мероприятия

## Отключение обработки

Для отключения обработки файлов и использования стандартного хранилища Yandex Cloud:

1. В `core/apps.py` закомментируйте вызов `self.apply_storage_backends()`
2. Или установите `USE_YANDEX_CLOUD=False` в `.env`

## Локальная разработка

Для локальной разработки используйте:

```bash
USE_YANDEX_CLOUD=False
```

Файлы будут сохраняться в директорию `media/`.

## Мониторинг и логирование

Все ошибки обработки логируются через стандартную систему логирования Django:

```python
import logging
logger = logging.getLogger(__name__)
```

Проверьте логи в файле `debug.log` для диагностики проблем.

## Очистка временных файлов

Временные файлы автоматически удаляются после обработки. Для ручной очистки:

```bash
rm -rf media_temp/*
```

## Производительность

Обработка файлов может занимать время:
- Изображения: 0.5-2 секунды
- Видео: зависит от размера и длительности

Для улучшения производительности:
- Используйте Celery для асинхронной обработки
- Настройте кэширование в Yandex Cloud CDN
- Оптимизируйте исходные файлы перед загрузкой

## Безопасность

1. Временные файлы хранятся только во время обработки
2. Доступ к `media_temp/` должен быть закрыт от публичного доступа
3. Проверьте права доступа к файлам водяного знака

## Устранение проблем

### Ошибка: "Файл водяного знака не найден"

Проверьте наличие файла по пути `media/watermark.png`

### Ошибка: "Не удалось сжать видео"

Проверьте установку зависимостей:
```bash
apt-get install ffmpeg
```

### Ошибка: "Временный файл не удален"

Проверьте права доступа к директории `media_temp/`

### Медленная обработка

- Увеличьте количество worker-ов Celery
- Используйте более мощный сервер
- Оптимизируйте исходные файлы
